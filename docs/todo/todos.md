## 2. Agent 事件决策逻辑

agent 如何决定什么内容构成一个「新事件」？如何在一段对话或一次操作中，为所有写入内容准确维护多事件标签？

### 问题
- 当前 L1（事件）的粒度由谁决定？agent 自行判断还是用户指定？
- 一段操作可能同时属于多个事件（如"安装 git"同时属于"工具链搭建"），如何自动为 L0 对话记录打上多事件标签？
- 事件与事件的层级关系（parent_event）如何自动推导？

### 思考方向
- 参考 AGENTS.md 的对话流程：对话开始时读取 L3 persona + 按需搜索上下文，对话结束时归档 L0 + 提取 L1
- L1 提取的时机在对话结束时，此时 context 最完整
- 可能需要设计一个"事件匹配规则"：根据对话中的关键词、项目引用、工具操作等自动判断归属事件
- 多事件标签意味着一个 L0 记录可以关联多个 L1 事件，允许跨事件搜索和聚合
